# Milestones

## Milestone 0

Analizar las historias de usuario y definir los objetos iniciales que formarán parte y se tendrán en cuenta en la lógica de la aplicación (esquema sustantivo -> objeto).

+ **Mínimo viable**

- Se define un objeto y/o atributos por cada aspecto a tener en cuenta en las historias de usuario (enfermero, contrato, turno, días libres...).
- Crear estructuras de datos adecuadas y almacenar información de ejemplo.

+ **Historias de usuario asignadas**

- HU1, HU2, HU3, HU4

## Milestone 1:

Asignación inicial de los horarios de cada enfermero siguiendo las condiciones de cada contrato, de forma que se emplee como base para la lógica a aplicar sobre el mismo.

+ **Mínimo viable**

- Se emplea la información obtenida en el milestone 0 para determinar los turnos en el horario.
- No hay puestos vacíos en ningún turno.
- Se cumplen todos los aspectos de los contratos

## Milestone 2:

Crear un cronograma visual interactivo con los horarios creados anteriormente.
